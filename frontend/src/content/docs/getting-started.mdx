# Getting Started

## Prerequisites

- **Node.js** >= 22
- **pnpm** >= 10
- **.NET 10 SDK** (for backend and Aspire orchestration)
- **Docker** (Aspire manages Postgres via Docker automatically)

## Quick Start (Recommended)

The fastest way to get the full stack running is with .NET Aspire. One command spins up Postgres, the .NET API, and the Astro frontend:

```bash
# clone and install
git clone https://github.com/codywilliamson/shockstack.git my-project
cd my-project
pnpm install

# build design tokens (required on first run)
pnpm tokens:build

# start everything with aspire
dotnet run --project backend/src/ShockStack.AppHost
```

Aspire will:
- Start a **Postgres** container with pgAdmin
- Launch the **.NET API** with service discovery and health checks
- Start the **Astro frontend** dev server
- Open the **Aspire dashboard** at `http://localhost:15888` for logs, traces, and metrics

## Frontend-Only Development

If you don't need the backend, you can run just the frontend:

```bash
pnpm install
pnpm tokens:build
pnpm --filter frontend dev
```

The frontend will start at `http://localhost:4321`. Auth and API features will gracefully degrade when the backend isn't available.

## Project Structure

```
shockstack/
├── frontend/                          # astro 5 application
├── backend/
│   └── src/
│       ├── ShockStack.Api/            # .NET 10 web API
│       ├── ShockStack.AppHost/        # aspire orchestrator
│       └── ShockStack.ServiceDefaults/ # shared service config
├── packages/tokens/                   # design token system (style dictionary)
├── docker/                            # docker compose configs
└── docs/                              # project documentation
```
